# 專案起始新手包 · 全域規範決策精靈

> 用問答的方式，幫非技術 PM 把網站專案的「全域規格」一次決定好——答完直接產出一份 Markdown 規格文件。

一個**單檔、零安裝、免註冊**的網頁小工具。打開就能用，所有答案只存在你自己的瀏覽器，不會上傳到任何伺服器。

👉 **線上使用：https://shellsirius.github.io/project-starter-kit/**

---

## 這是什麼

開新的網站／系統專案時，有一堆「全站共通」卻最常被漏掉的決定：

- 頁首頁尾要放哪些資訊？要不要自動登出？倒數幾分鐘？
- 密碼規範、錯誤鎖定、Token／Session 效期怎麼定？
- 要不要無障礙？哪個等級？多語系？在地化格式？
- 通知怎麼寄、要不要稽核軌跡、個資怎麼處理？
- 資安檢測、機房部署、技術選型……

這個精靈把這些問題整理成 **6 個關卡**，一關一關帶你選，邊填邊在右側即時產出結構化的 Markdown，最後複製或下載，貼進你的 SRS／SA 或交給工程團隊。

## 給誰用

非技術的 **PM、專案窗口**。不用懂技術，照著選就好；不確定的題目可以先跳過，最後文件會自動標出「尚未決定」的項目。

## 怎麼用

1. 下載或 clone 本專案，用瀏覽器打開 `index.html`（雙擊即可，不需要安裝任何東西）。
2. 一關一關回答問題，右側即時看到產出的規格文件。
3. 按「複製 Markdown」或「下載 .md」，把規格帶進你的文件流程。

> 上方的 **🔧 技術題** 開關可一鍵隱藏工程／合約層的題目，只留 PM 能決策的項目。

## 隱私

你的答案只會存在**這台電腦的瀏覽器**裡（localStorage），不會上傳到任何伺服器，也沒有任何追蹤。換電腦或清除瀏覽器資料後即消失。

## 給團隊：自訂預設值

公開版預設**中立**、不預選任何技術選型。團隊若想預載自家慣用值（例如固定的前端框架、資料庫、版控），打開 `index.html`，找到最上方的 `PRESET` 物件，把預設答案填進 `defaults`：

```js
const PRESET = {
  name: '我的團隊',
  defaults: {
    fe:  'Vue',      // 前端
    be:  '.NET',     // 後端
    db:  'PostgreSQL',
    vcs: 'GitHub'
    // 值需為該題在 CONFIG 裡既有的選項字串
  }
};
```

題庫的完整結構與分支邏輯，see [`題庫大綱-v1.md`](題庫大綱-v1.md)。要改題目，直接編輯 `index.html` 裡的 `CONFIG` 陣列即可（同步更新題庫大綱）。

## 技術

- 單一 `index.html`，內含 HTML／CSS／JS，**無建置流程、無相依套件**。
- 字體用 Google Fonts（Zen Maru Gothic／Noto Sans TC／DM Mono）。
- 可直接部署到 GitHub Pages、Vercel、Netlify 等任何靜態空間。

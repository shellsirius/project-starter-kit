> 用途：給非技術 PM 的「全域規範問答精靈」題庫。
> 本檔與 `index.html` 的 `CONFIG` 同步；改題庫時兩邊一起更新。
>
> 標記：🔧 技術/合約層（關卡五～六，可整關隱藏）｜【推薦】預設建議選項｜【選填】不計入完成數｜↳ 分支邏輯｜💡 為什麼問/差異說明

> 架構原則：PM 能決策的（關卡一～四）在前，技術/合約（關卡五資安與部署、關卡六開發環境）收最後；關閉「技術題 🔧」開關即隱藏整個關卡五、六。

## 關卡一　系統定位
> 系統是什麼、給誰用、跑在哪些裝置與語言環境。

### 1-1　基本設定
- **專案系統類型**：業務管理後台 / 民眾網前台 / 獨立儀表板 / Kiosk
  - ↳ 選「民眾網前台」→ 解鎖關卡二的 SEO、Cookie 題
- **適用裝置 (RWD)**：電腦尺寸即可 / 全尺寸響應式 / 特定尺寸（↳ 自填尺寸）
  - ↳ 選「電腦尺寸即可」→ **最低支援視窗寬度**：1024 / 1280 / 1366 / 1440 / 1920 px / 其他
    - 💡 1366px 最常見筆電解析度、1920px 為 Full HD
- **支援瀏覽器清單**（複選）：Chrome【推薦】/ Edge【推薦】/ Safari / Firefox / IE 相容
- **是否需要無障礙**：不需要 / 是
  - ↳ 選「是」→ **等級**：A / AA【推薦】/ AAA
    - 💡 三級累加（AA 含 A、AAA 含 AA）。A=基本門檻；AA=好用（對比 4.5:1、放大 200% 不破版、focus 可見）為政府/法規標準；AAA=最嚴（對比 7:1、手語影片、無時限），整站難全達成
- **是否需要 GA 分析**：不需要 / 是（↳ 追蹤碼 G-XXXXXXXX）

### 1-2　語言與在地化
- **是否需要多語系**：不需要 / 是
  - 💡 多語系獨立一個需求編號；翻譯由誰做、翻譯社、實作方式留訪談寫細節
  - ↳ 選「是」→ **支援語言**（複選）：繁體中文【推薦】/ 英文【推薦】/ 日文 / 韓文
  - ↳ 選「是」→ **其他語言**【選填】
- **日期紀年方式**：西元 / 民國 / 其他（↳ 自填）
- **時區**：GMT+8（台北）【推薦】/ 依使用者所在時區 / 其他（↳ 自填）
- **金額／數字格式**：千分位逗號（1,234,567）/ 無分隔 / 其他（↳ 自填）
- **時間顯示制**：24 小時制【推薦】/ 12 小時制（AM／PM）

## 關卡三　帳號與登入
> 登入、密碼、權限與 Token／Session 行為。

### 3-1　登入設定
- **帳號建立方式**：開放使用者自行註冊（↳ 是否需審核）/ 僅管理者後台建立 / 兩者皆可
- **SSO／單一登入整合**：不需要 / 需要（↳ 整合對象，例 AD、LDAP、政府公務帳號、OAuth）
- **是否需要圖形驗證碼**：是 / 否（↳ 樣式：6 碼數字 / 6 碼英數 / 6 碼英文 / reCAPTCHA / 其他）
- **是否需要忘記密碼功能**：是 / 否
- **是否需要雙因素驗證（2FA／OTP）**：不需要 / 需要（↳ 方式）
- **初次登入是否強制變更密碼**：是 / 否

### 3-2　錯誤鎖定
- **密碼錯誤是否需鎖定**：是 / 否（↳ 選「是」才展開以下）
  - ↳ **錯誤次數上限**：3 / 5 / 10 次 / 其他
  - ↳ **鎖定時間**：5 / 15 / 30 分鐘 / 其他
  - ↳ 選「是」才解鎖下方鎖定模式
- **鎖定模式**：鎖定帳號【推薦】/ 鎖定當前瀏覽器
  - 💡 鎖定帳號＝DB 記錄，換瀏覽器/重整/換裝置都無法解除（注意可能被惡意鎖死）；鎖定瀏覽器＝重啟即重置，較寬鬆但防護弱。政府/高安多選前者

### 3-3　密碼規範與效期
- **密碼長度（字元數）**：自填，預設 `8-20`
- **密碼規範**（複選）：必須包含數字 / 必須包含英文大寫 / 必須包含英文小寫 / 必須包含特殊符號 `#?!@$%^&*-`
- **輸入字元限制**【選填】：不可輸入＿＿ / 僅限輸入＿＿
- **變更密碼不可與前幾次相同**：不需要 / 有（↳ 3 / 5 / 6 次 / 其他）
- **密碼最長效期**：不需要 / 有（↳ 30 / 60 / 90 / 180 / 365 天 / 其他）
  - 💡 密碼用超過設定天數就強制更換，定期汰換舊密碼、降低長期外洩風險
  - ↳ 若「不可與前幾次相同＝不需要」且「最長效期＝有」→ **是否提供沿用舊密碼**：是 / 否（💡 到期被迫換密碼時，可否直接沿用原密碼）
- **密碼最短效期**：不需要 / 有（↳ 1 / 2 / 3 天 / 其他）
  - 💡 改密碼後至少隔幾天才能再改，防止使用者連續快速改密碼把舊密碼換回來（規避「不可重複」規則）

### 3-4　權限設定
- 可否變更他人密碼：是 / 否
- 可否變更自己密碼：是 / 否
- 可否刪除／停用自己帳號：是 / 否
- 可否刪除／停用自己帳號的權限角色：是 / 否

### 3-5　Token / Session
- **Token 效期**：1hr / 8hr / 24hr / 7 天 / 其他
- **Token 更新**：是 / 否
  - 💡 Token＝登入後系統發給瀏覽器的「通行證」，每次操作靠它證明已登入。效期＝多久過期（過期需重新登入）；Token 更新＝是否在快到期時自動續一張新的，避免持續操作時突然被登出
- **Token 失效檢查方案**（複選）：一段時間主動檢查 / 動作檢查 / 進入頁面檢查
  - 💡 主動檢查＝最即時但耗資源；動作檢查＝省資源但閒置不察覺；進入頁面＝最省但停同頁不察覺。可併用
- **關閉瀏覽器是否保持登入**：是 / 否
- **同瀏覽器允許分頁登入不同帳號**：是 / 否
- **同一帳號是否限制單一登入**：是（新登入踢掉舊的）/ 否
  - 💡 同帳號能否同時在多裝置/瀏覽器登入。是＝只能一處，他處登入會踢掉前一個；否＝可多處同時。常用於防帳號共用或高安系統
- **自動登出倒數重置行為**（複選）：click【推薦】/ contextmenu【推薦】/ keydown【推薦】/ mousemove / scroll
  - 💡 mousemove、scroll 觸發極頻繁，可能效能問題；建議只開前三

## 關卡四　資料與通知
> 通知寄送與資料治理等功能性全域決策（多為原清單常漏項）。

### 4-1　通知寄送
- **是否需要系統通知**：需要 / 不需要（↳ 選「需要」才展開以下）
- **通知管道**（複選）：Email / 簡訊 / 站內訊息
- **寄信方式**：客戶 SMTP / 第三方（SendGrid 等）/ 待定（↳ 通知管道含 Email 才出現）
- **觸發情境**：自填（例：註冊、忘記密碼、審核結果、到期提醒）

### 4-2　資料治理
- **操作稽核軌跡 (audit log)**：需要【推薦】/ 不需要
  - 💡 政府案幾乎都要求操作軌跡
  - ↳ 選「需要」→ **記錄哪些動作**（複選）：登入／登出【推薦】/ 新增【推薦】/ 修改【推薦】/ 刪除【推薦】/ 查詢／檢視 / 匯出／下載 / 審核／核可 / 權限／帳號異動；另有「其他」【選填】
- **資料保存期限 / 自動清除**：有（↳ 期限）/ 無
- **資料匯出**：需要（↳ 格式 Excel/CSV/PDF）/ 不需要
- **是否含個人資料（個資）**：有個資（↳ 去識別／刪除機制）/ 無
  - 💡 個資＝可識別特定個人的資料（姓名、身分證、電話、Email…）。有的話依個資法常需提供當事人刪除權／去識別機制

## 變更紀錄
- 重新分類為 6 關卡（PM 行為層關卡一～四前置；技術/合約層收於關卡五資安與部署、關卡六開發環境），題組重排、關聯題集中。
- 開發環境獨立為關卡六；關卡五更名「資安與部署」。
- 在地化拆為日期紀年方式／時區／金額／時間顯示制各一題；多數數值欄位改「常見選項＋其他」。
- 版本號顯示重構：是否顯示版本號（總開關）→ 顯示位置複選（頁首/頁尾/登入頁/選單）＋其他位置。
- 新增：支援瀏覽器、麵包屑、帳號建立方式、SSO、2FA、單一登入、通知寄送、資料治理、SEO/Cookie/維護頁/錯誤頁面客製。
- 分支連動：語言切換僅多語系=是時出現；登入頁版本號併入版本號位置；SEO/Cookie 依民眾網前台帶出。
- 標示推薦選項；補無障礙等級、登入鎖定、Token 檢查、備份/備援差異說明。
- 「其他語言」「輸入字元限制」「其他位置」設為【選填】，不計入完成數。
- 去鎰威化（v1.1）：新增 PRESET 預設層；關卡六技術選型中立化；範例文字移除鎰威／特定客戶名；移除「建議新增」標籤（屬內部對照舊清單用，對公開使用者無意義）。
